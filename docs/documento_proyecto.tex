\section{Descripción del Proyecto}

\subsection{Contexto y Problemática}

En la actualidad, la asistencia a eventos culturales, deportivos y de
entretenimiento sigue dependiendo de boletos físicos impresos. Este
modelo presenta inconvenientes como pérdida del boleto, dificultad
para transferirlo y procesos de cancelación lentos.

\textbf{EventTix} propone reemplazar el boleto físico por un
\textit{Smart Ticket} digital con código QR, accesible desde cualquier
dispositivo Android, eliminando la necesidad de impresión y mejorando
la seguridad en el control de acceso.

\subsection{Objetivo General}

Desarrollar una aplicación móvil en Flutter que permita a los usuarios
explorar eventos, gestionar sus boletos digitales y acceder a ellos
mediante códigos QR únicos.

\subsection{Objetivos Específicos}

\begin{itemize}[noitemsep]
    \item Diseñar una cartelera de eventos interactiva con datos estáticos (Fase 1) y REST API (Fase 2).
    \item Implementar una billetera digital que centralice todos los boletos del usuario.
    \item Generar códigos QR únicos para cada boleto usando la librería \texttt{qr\_flutter}.
    \item Permitir la transferencia y cancelación de boletos desde la aplicación.
\end{itemize}

\section{Funcionalidades Principales}

\subsection{Pantalla 1 -- Cartelera de Eventos}

Lista de próximos eventos con tarjetas visuales. Cada tarjeta muestra
nombre, lugar, fecha, categoría y precio. En esta fase los datos son
estáticos (\textit{mock data}); en Fase 2 se conectará a una API REST.

\begin{itemize}[noitemsep]
    \item Vista en lista con \texttt{ListView.builder}.
    \item Categorías: Música, Comedia, Danza, Electrónica.
    \item Indicador de precio por entrada.
\end{itemize}

\subsection{Pantalla 2 -- Billetera de Entradas}

Sección personal que centraliza todos los boletos del usuario,
diferenciando visualmente los \textbf{Activos} de los
\textbf{Cancelados}. Toca un boleto activo para ver su código QR.

\begin{itemize}[noitemsep]
    \item Lista con nombre del evento, asiento y fecha.
    \item Indicador de estado: ACTIVO / CANCELADO.
    \item Navegación directa al detalle del boleto.
\end{itemize}

\subsection{Pantalla 3 -- Smart Ticket (Código QR)}

Pantalla central de la aplicación. Muestra la información del boleto
y genera un código QR real a partir del ID del boleto, evento, asiento
y fecha.

\begin{itemize}[noitemsep]
    \item QR generado con \texttt{qr\_flutter \^{}4.1.0}.
    \item Diseño tipo boleto con separador troquelado.
    \item Botones \textbf{Transferir} y \textbf{Cancelar} (Fase 2 -- backend).
\end{itemize}
